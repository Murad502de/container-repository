# Requirements

This folder stores requirements documents owned by this container boundary.

## What Belongs Here

- Functional requirements owned by this container boundary.
- Non-functional requirements and acceptance criteria.
- Business rules that shape system behavior across repositories.

## What Does Not Belong Here

- Architecture decisions. Use [../../adr](../../adr/README.md).
- System or technical design. Use [../system-design](../system-design/README.md) or [../technical-design](../technical-design/README.md).
- API payload contracts. Use [../../contracts](../../contracts/README.md).

Use [requirements-template.md](../../templates/requirements-template.md).
